# Azureポータルでのリソース作成手順(ACS)

`brain/`(脳)がACSユーザーの作成・トークン発行を行うために必要な、Azure Communication Services (ACS) リソースの作成手順。

## 前提

- Azureサブスクリプションへのアクセス権(リソース作成が可能なロール)
- ACSリソースを作成するテナントは、対象Teams会議のテナントと同じ(または信頼関係がある)こと

## 手順

1. [Azureポータル](https://portal.azure.com/)にサインインする
2. 「リソースの作成」→ 「Communication Services」を検索して選択する
3. 以下を入力してリソースを作成する
   - **サブスクリプション**: 対象のサブスクリプション
   - **リソースグループ**: 新規作成、または既存のものを選択
   - **リソース名**: 任意(例: `teams-scriber-acs`)
   - **データの場所**: Teams会議参加者の地域に近いリージョンを選択(例: Japan)
4. 「確認と作成」→「作成」でデプロイを実行する
5. デプロイ完了後、リソースに移動し、左メニューの **[キー]** を開く
6. **接続文字列(Connection string)** をコピーする
   - これが `ACS_CONNECTION_STRING` 環境変数の値になる
   - Primary / Secondary どちらでもよいが、ローテーション予定がある方を避けるとPoC運用中の失効を避けやすい

## 動作確認

接続文字列を発行したら、`brain/` から以下を実行してACSユーザー作成・トークン発行が成功するか確認する。

```sh
cd brain
uv sync
ACS_CONNECTION_STRING="<コピーした接続文字列>" uv run issue-acs-token
```

`{"user_id": "...", "token": "...", "expires_on": "..."}` がJSONで標準出力に表示されれば成功。

## 注意点

- 接続文字列はシークレットなので、`.env` やCI変数に保存し、リポジトリにはコミットしないこと(`.gitignore` で `.env*` を除外済み)
- ACSの無料利用枠を超えると課金が発生する。PoC期間中は使用量を定期的に確認すること
- 会議のテナント側で **キャプション/トランスクリプションポリシー** および **外部ユーザーのロビー通過・ボット検出ポリシー** が有効になっているか、あわせてTeams管理センターで確認すること(ACS側の設定ではないため、この手順には含まれない)
